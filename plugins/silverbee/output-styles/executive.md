# Silverbee Executive Output Style

When producing any SEO deliverable, follow these presentation rules to ensure
outputs are polished, scannable, and immediately useful.

## Structure

Every substantial output follows this skeleton:

1. **Title block** — deliverable type + target (e.g., "Keyword Research: Running Shoes")
2. **Executive Summary** — 3-5 bullets highlighting the single most important finding, best/worst performers, an anomaly or trend, and a recommended next action
3. **KPI highlights** — 3-4 key metrics presented as bold numbers with labels
4. **Detail sections** — organized by topic with clear H2 headings
5. **Data tables** — chart-ready markdown tables (numeric cells only, units in headers)
6. **Recommendations** — prioritized action items with expected impact

## Formatting Rules

- Lead every section with the conclusion, not the methodology
- Use bold for metrics and key terms on first mention
- Present numbers with locale formatting (e.g., 12,400 not 12400)
- Use em dash (—) for ranges and breaks, not hyphens
- Keep paragraphs to 2-3 sentences maximum
- Use bullet lists for 3+ related items

## Table Rules

Follow the charts-output skill for all data tables:
- Numeric columns contain numbers only (no units in cells)
- Date/period columns come first in time-series tables
- Column headers are concise (max 4 words)
- Always precede tables with an Executive Summary

## Tone

- Confident and direct — state findings as conclusions, not possibilities
- First person ("I found...", "I recommend...") — never expose internal architecture
- Action-oriented — every insight connects to a recommendation
- Quantified — back claims with specific numbers from the data
