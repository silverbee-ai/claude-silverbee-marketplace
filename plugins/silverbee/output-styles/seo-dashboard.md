# SEO Dashboard Output Style

When completing any SEO analysis, research, or audit, **always produce two outputs**:

1. An inline interactive dashboard via `show_generative_ui`
2. A self-contained HTML report file written to disk

Never skip either output. Both must be produced before adding follow-up text.

---

## Output 1 — Inline Interactive Dashboard

Call `show_generative_ui` with a Stack spec immediately after all data is collected.
Load `react-wow-output` skill for the component API reference.

Hard limits:
- Exactly **4 Metric** components
- **1 Chart** maximum
- **2 Tables** maximum, combined **≤ 12 rows**
- Title format: `"[Workflow] — [Target] ([Market])"`

---

## Output 2 — HTML Report File

After `show_generative_ui`, write the HTML report using the Write tool.

**File path:** `[cwd]/[domain-slug]-[workflow-slug]-[YYYY-MM-DD].html`
Example: `bkzlaw-keyword-research-2026-03-31.html`

**How to produce the file:**
1. Copy the HTML template below verbatim
2. Fill in only the `REPORT` JavaScript object with real data
3. Do not modify anything outside the `REPORT` object

---

## Output Sequence

This style is referenced by `seo-output-formatter`. When using it standalone:

1. Complete all data collection
2. Call `show_generative_ui` with the Stack spec (Layer 1)
3. Write the HTML file using the Write tool (Layer 2)
4. Write the full markdown deliverable in chat — complete tables, no row cap (Layer 3)
5. Write a single next recommendation (≤ 50 words, statement format, no questions)
